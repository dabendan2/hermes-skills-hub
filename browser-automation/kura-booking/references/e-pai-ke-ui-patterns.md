# E-Pai-Ke UI 交互模式

## 核心 UI 組件
- **分店 ID**: 
  - 土城金城路店: `90510`
  - 中和橋和店: `90342`
  - 板橋遠科店: `90506`
- **預約 Modal**: ID 通常為 `#orderModal`。
- **日期欄位**: `#reserve_date`。

## 用戶偏好與流程修正
- **直接操作**: 當用戶提供帳密時，停止搜尋 `.md` 檔案，直接將資訊寫入 `.env` 並執行 Playwright 腳本。
- **重置邏輯**: 若點選日期無反應，用戶期望執行「關閉 -> 重新開啟」動作。這能處理前端 JS 狀態未更新的問題。
- **證據優先**: 在報告「額滿」前，必須有截圖證明（如 `kura_select_day7_proof.png`），並在截圖中標示點擊位置（如用 JS 畫紅框）。

## 狀態代碼參考
| 狀態 | Class/現象 | 意義 |
|------|-----------|------|
| 可預約 | `dateOk` | 營業中且有名額 |
| 額滿 | `dateNo` | 當日線上預約已滿 |
| 鎖定 | `ui-state-disabled` | 系統不開放或日期已過 |
| 無時間 | 下方顯示「請先點選此處...」 | 未選中有效日期或時段已滿 |
