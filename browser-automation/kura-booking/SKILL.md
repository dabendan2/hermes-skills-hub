# E-Pai-Ke (E排客) Skill

本 Skill 用於透過 Playwright 腳本自動化操作 E排客 (e-pai-ke.com) 進行各分店餐廳的預約、查詢及取消。

## [重要規範 (MUST READ)]
1. **單一預約限制**：同一帳號在同一家分店**絕對只能擁有一筆**有效預約。
    - **換日/更改預約流程**：若要更改同一分店的預約日期或時間，**必須先執行 `cancel.js` 取消現有預約**，否則系統會阻擋新預約。
    - **禁止盲目嘗試**：不要在未取消舊預約的情況下執行 `book.js`，這會觸發腳本內的重複判定邏輯並自動停止任務。

## 帳號資訊參考
- 登入資訊記載於 `.env` 檔案中的 `E_PAI_KE_EMAIL` 與 `E_PAI_KE_PASSWORD`。

## 操作流程 (Playwright 腳本)

本技能完全依賴 Playwright 腳本執行。**禁止使用標準瀏覽器工具 (browser_navigate 等) 手動操作**，因為該網站在內建工具下極不穩定且容易逾時。

### 1. 預約腳本 (`scripts/book.js`)
**使用方法：**
```bash
node ~/.hermes/skills/kura-booking/scripts/book.js <店名或ID> <日期數字> <時間格式> <人數>
```
- **參數格式規範**：
    - `店名或ID`: 分店名稱（如 `土城金城店`）或 5 位數 ID（如 `90510`）。
    - `日期數字`: **純數字**。不帶月、不帶「號」。(範例: `7`, `15`)
    - `時間格式`: **HH_MM**。使用底線分隔。(範例: `18_30`, `11_00`)
    - `人數`: **純數字**。(範例: `2`)

---

### 2. 取消預約腳本 (`scripts/cancel.js`)
**使用方法：**
```bash
node ~/.hermes/skills/kura-booking/scripts/cancel.js "<月份字串>" "<日期字串>"
```
- **參數格式規範 (極重要)**：
    - `<月份字串>`: 格式必須為 **MM月**。包含「月」字，且 10 月以下需補 0。(正確範例: `05月`, `11月`)
    - `<日期字串>`: 格式必須為 **DD**。10 日以下需補 0。(正確範例: `07`, `15`)

**快速範例**：若要取消 5月 7號 的預約：
`node cancel.js "05月" "07"`

## 避坑指南 (Pitfalls)
1. **不要搜尋特定測試檔案**: 執行任務時，優先使用腳本直接操作。不要花時間搜尋過往對話中的截圖。
2. **獲取 Shop ID**: 若不知道 ID，導航至 `https://e-pai-ke.com/search?keyword=店名` 獲取。
3. **重複預約攔截**: 腳本內建「已完成候位」偵測。

## 關鍵技術細節 (Lessons Learned)
1. **延遲載入 (10s)**: 開啟預約彈窗後必須等待至少 10 秒。
2. **時段 ID 格式**: 系統 ID 為 `#hour_HH_min_MM`。腳本已自動處理。
3. **取消確認**: 取消包含兩階段，必須點擊 `#delOK`。
4. **UI 元素拆分**: 月份與日期在 HTML 中是分離的。

## 錯誤偵測與反饋
- **日期不可選**: 日曆標籤帶有 `dateNo` 類別即代表額滿。
- **時段不可選 (FULL)**: 時段按鈕帶有 `disabled` 屬性或 `dateNo` 類別即代表該時段已額滿。
    - **改進**: `book.js` 已加入自動偵測。若目標時段額滿，腳本會立即報錯並主動抓取當天「剩餘可預約」的時段供參考。
- **時段缺失**: 若點擊日期後 10 秒仍無時段按鈕，代表該日已完全額滿。
